## Homelab Baremetal Provisioning

In my latest homelab project, I wanted to move away from manual "button-clicking" and toward a truly automated infrastructure. The goal was simple: **Zero-touch bare metal provisioning.** I wanted to be able to plug in a node and have it join a production-ready Kubernetes cluster without ever manually running an installer.

Here is how I layered **Tinkerbell**, **Talos Linux**, **Ansible**, and **Flux CD** to make it happen.

### The Architecture: A Four-Layer Cake

My workflow is broken down into distinct stages of responsibility, ensuring that if one layer fails, I know exactly where to look.

#### 1. The Provisioning Layer (Tinkerbell & Ansible)
Everything starts on my **Manager Node** (Ubuntu). I use **Ansible** as the primary orchestrator. It manages my **Tinkerbell** workflows, which handle the PXE booting and image distribution. When a new node (like my Raspberry Pis or x86 nodes) hits the network, Tinkerbell recognizes the MAC address and begins the automated Talos installation.

#### 2. The Operating System (Talos Linux)
I chose **Talos Linux** for the cluster nodes specifically because it is immutable and "headless." There is no SSH and no shell—only an API. This reduces the attack surface and ensures that the OS state is consistent across my control plane and workers. Ansible handles the `talosctl apply-config` and initial bootstrap to get the cluster talking.

#### 3. The GitOps Layer (Flux CD)
Once the cluster is alive, Ansible performs a `flux bootstrap`. This is where the "magic" happens. Flux points to my [GitHub repository](https://github.com/darrylbalderas/homelab-k8s) and begins a multi-stage reconciliation:
* **Infrastructure:** Setting up the "must-haves" like **Cilium CNI**, **MetalLB**, and **Longhorn** for storage.
* **Infrastructure-Config:** Applying the actual Gateway classes and IP pools.
* **Platform:** Deploying my observability stack (**Prometheus, Loki, Tempo**) and **Dapr**.

#### 4. The Networking & Service Layer
By using the **Envoy Gateway** and **Gateway API CRDs**, I’ve moved toward a more modern way of handling traffic than traditional Ingress. Combined with **Cilium’s** eBPF-powered networking, the cluster is highly performant and secure.

### Why this matters
Building this wasn't just about having a cluster—it was about building a **resilient system**. Because the Talos nodes are stateless, I can lose a machine, re-provision it via Tinkerbell, and Flux will automatically bring all the networking and apps back to the desired state within minutes.
